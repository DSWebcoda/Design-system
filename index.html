Button System

Our button system provides six type variants with three states each (enabled, hover, disabled), three size variations (large, medium, small), and three icon configurations (no icon, icon right, icon left) — matching the Figma design specifications exactly.

Sizes

  • Large: 48px height
  • Medium: 40px height
  • Small: 32px height

Button Icon Configurations

Buttons support three icon configurations: no icon, icon right (arrow forward), and icon left (arrow back). Icons should be 16px for small buttons, 20px for medium, and 24px for large buttons.

Important: Add btn-icon-right or btn-icon-left class to buttons with icons. This reduces padding on the icon's side to half, creating better visual balance.

Usage Guidelines

Primary Turquoise

Use for primary actions and main call-to-actions. High contrast and attention-grabbing. Perfect for key conversion actions.

Primary Sunshine

Use for accent actions and warm call-to-actions. Great for book/purchase actions and secondary CTAs.

Secondary on Light

Use on light backgrounds for secondary actions and navigation. Provides clear contrast on white/light surfaces.

Secondary on Dark

Use on dark backgrounds like headers and overlays. Maintains visibility and hierarchy on dark surfaces.

Tertiary

Use for subtle, low-emphasis actions. Perfect for cancel, dismiss, or optional actions that shouldn't dominate the interface.

Overlay

Use over images and complex backgrounds where transparency is needed. Ensures button visibility on varied backgrounds.

Usage Guidelines

Single Selection

Only one date can be selected at a time. Selecting a new date will deselect the previously selected date.

Month Navigation

Use the left and right arrow buttons to navigate between months. The calendar supports both past and future dates.

States

Usage Guidelines

When to Use

Use accordions for organizing related content that doesn't need to be viewed all at once. Great for FAQs, feature lists, or detailed information that users might want to scan.

Accessibility

Accordion headers are buttons and can be navigated using keyboard. Use proper ARIA attributes and ensure content is accessible to screen readers.

Content Length

Keep accordion titles concise but descriptive. Content inside can be longer but should still be scannable.

Icon System

A comprehensive collection of Material Design icons for user interface elements. All icons are SVG-based and support theme colors through CSS variables.

Available Icons

Icon Guidelines

Icons should be used consistently throughout the interface. Use semantic icon names when possible and maintain consistent sizing across similar interface elements.

Sizing

Standard icon sizes are 16px, 20px, 24px, and 32px. Use appropriate sizes based on context and visual hierarchy requirements.

Implementation

Reference icons using relative paths: icons/material-symbols_[name].svg. SVG files can be embedded inline or referenced as images.
